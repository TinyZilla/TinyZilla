## Past Experiences

I've been in multiple work environments. From start ups, mid-size to corporate, I've seen and understand the requirements and challenges for the majority of the scenarios that you'll encounter.

### Booking Software @ CTRL V
CTRL V is a VR Arcade chain looking to create its own booking system to move away from its existing one created for movie theaters. It was a very "start up" experience for me. I was given projects that I had no prior experience to, relying on docs, stack overflow posts, and existing code from previous interns for the majority of the systems I've built.

I ended up designing a lot of the system. Things like:
- Booking flow
- Spot reservation before checkout
- Integration with Payment Processor
- Cart Calculation
- Gift Card System
- API Interface on both Frontend and Backend
- CSS Styling / Little Animations

This is definitely a more "trail by fire" experience for me. I wore many hats and learned a lot from each responsibility. The most important thing I've learned is that I have the capability to pick up and absorb new information and tech very quickly.

The project has a lot that could be improved on, but it is still working well and serving the customer till this day. You can check it out [here](https://www.ctrlv.ca/booking/location).

### Internal Tooling @ Ritual.co

Ritual is a food ordering platform aimed at metropolitan office workers. I was there during its growth phase. The team I was part of is in charge of building internal tools and dashboards to manage merchants and users alike.


This was the first time I worked in a team environment, with the team lead planning and creating tasks to assign to everyone else. I've learned a lot about testing, authentication, middlewares, and architecture here.


With a team there's a lot more communication and accountability that is in the day to day. I got a taste of working with different departments such as QA and UI/UX Designers here.

This is also a place where I was exposed to DevOps tools and processes such as Docker, CI/CD, github actions, and cloud infrastructures. Thanks to those experiences I've had the interest to pick up a VPS and hosted various services for my own personal use.

### Mobile Game Services @ Kabam

I was part of the Marvel Contest of Champions's backend team, creating backend services to handle new features and progression systems. During my time there, I was part of the team that shipped:
- Relics -- equipment system for character to boost & alter gameplay
- Ascension -- power up system for older character to boost stats
- 7 Stat Focus -- unique mechanic for the new rarity of characters which boost player selected stats.


Since this is a 10+ year old game, the code base was difficult to work with. There was a lot of tech debt built up over the years, and a lot of modern features of JavaScript and MongoDB interfaces are not present in the system that we are interacting with. Furthermore due to the "go fast" mentality there wasn't a lot of thought put through when designing the API for the core systems that was built since the game's inception.


Through this experience I understood the importance of good architecture, and in lue of that, good documentation. During my time there I learned to leave behind flow charts and thought processes so that it'll help someone to understand the design and motivation behind the system.

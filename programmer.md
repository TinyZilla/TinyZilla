## Programming Languages
I am proficient in Rust and JavaScript. I've work with Java, Python, C# professionally before. I've handled C++ in university courses and assignments.

I believe programming skills is transferrable, so language doesn't matter beyond familarity. Over time I will have the same problem solving skills as I would have any other language. 

With that being said, I have a strong preference for Typed Programming Languages. After working with JavaScript for so long having some sort of expectation on the object you are receiving or sending is nice.

I also have a strong preference in functional programming. It's much easier to reason about when there's a logical separation between pure functions and functions that alters something.

## Programming Design
I like the concept of domain driven design. Modelling the structure of the software akin to how one would be describing it to another person in domain expert terms. This maps the concept one to one and there isn't that big of an overhead swapping between a problem domain and programming solution.

## Testing & Documentations
I believe unit testing is important for business logic. Functions which transform data should be captured in test suits. So that it can be serve as both a guarentee and documentation for future programmer who is looking to alter the code.

As for code involved for Data plumming such as Interfaces and wrappers, those can be just captured by integration tests.

## Stance on AI
I don't believe AI is currently sufficent to replace for human in Software Engineering. Although the goal of programmers is to produce useful artifacts that will bring value to the user / company, it's the understanding of the problems and their solution is what makes our services valueable. Although AI can produce code faster, but it often messes up the constraints or context that was implied if the task was given to a human. Until that capability is achieved, human supervision is still required.

As it is right now, AI is a very good research tool. It can condense a lot of information that was scattered and present it in a single location. You do need to double check the information source and verify if it works first, but nonetheless it eases a lot of friction and saves a decent amout of time.

Regardless of what happens in the future, AI shouldn't replace learning. Just like having a calculator doesn't eliminate the need to learn the concepts behind math, having AI shouldn't mean that we stop understanding coding and programming in general.
